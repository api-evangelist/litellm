LiteLLM introduced Lite-Harness SDK to address vendor lock-in at the harness layer, enabling developers to swap between Claude Code, Codex, and Pi AI by changing a single string. The SDK exposes all harnesses through a unified Claude Agents SDK specification, and works standalone with native provider APIs or alongside the LiteLLM AI Gateway for centralized key management, budgets, and fallbacks.
